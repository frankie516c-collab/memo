최가영 문제
```
/** 8) 문자 F의 인덱스값을 찾으시오.
 * 출력값 :
 * 인덱스 값 : (1,2)
 ****************/
const arr3 = [
  ['A','B','C'],
  ['D','E','F'],
  ['G','H','I']
];
```

지피티의 답
```
let 문자 = 'F';
let 답 = '';

for (let i = 0; i < arr3.length; i++) {        // 행(row)
  for (let j = 0; j < arr3[i].length; j++) {   // 열(column)
    if (arr3[i][j] === 문자) {
      답 = `(${i},${j})`;
    }
  }
}

console.log(`찾는 문자 : ${문자}, 인덱스 값 : ${답}`);
```

Q&A
- `let i = 0`부터 시작하는 이유: index(자리 번호)의 개념; 시작점에서 얼마나 떨어져있나를 의미함
  - 만약 1부터 시작한다면? arr3[0]을 보지 못하고 arr3[3]에도 접근을 하지 못할 것
  - 한 줄 요약: 배열은 "시작점에서의 거리"로 위치를 센다. 그래서 항상 0부터 시작한다.
 
- `arr3.length === 3`라는 선언 없이도 `arr3.length`로 행이 3개라는걸 알 수 있는 이유: 배열은 자기 길이를 스스로 알고 있는 상자
  - `.length`는 "미리 정해진 기능"
  - `arr3.length`라는거 자체가 이 배열 안에 몇 개가 들어 있는지를 JS에게 물어보는 것
  - 굳이 숫자 3을 쓰지 않고 `.length`를 쓰는 이유: 배열이 바뀌면 코드도 고쳐야하기 때문
  - 한 줄 요약: 배열은 자기 안에 몇개가 들어 있는지 처음부터 알고 있고 `.length`는 그걸 물어보는 질문이다.

- `arr3[i].length`를 사용하는 이유
  - `arr3`는 **배열 안에 배열이 들어있는 구조**라서 안쪽 배열(arr3[i])의 길이를 기준으로 반복해야 해.
 
아주 좋은 질문이야 👍
이걸 이해하면 **2차원 배열을 완전히 이해한 거야**.

---

## 핵심 이유 한 줄 요약

👉 `arr3`는 **배열 안에 배열이 들어있는 구조**라서
👉 **안쪽 배열(arr3[i])의 길이**를 기준으로 반복해야 해.

---

## arr3 구조부터 보자

```js
const arr3 = [
  ['A','B','C'],  // arr3[0]
  ['D','E','F'],  // arr3[1]
  ['G','H','I']   // arr3[2]
];
```

### 인덱스로 보면

* `arr3.length` → 3 (행 개수)
* `arr3[0].length` → 3 (첫 번째 행의 열 개수)
* `arr3[1].length` → 3
* `arr3[2].length` → 3

---

## 바깥 for문 vs 안쪽 for문

```js
for (let i = 0; i < arr3.length; i++) {
```

👉 **행(row)** 을 하나씩 이동
👉 `i = 0 → 1 → 2`

```js
for (let j = 0; j < arr3[i].length; j++) {
```

👉 **현재 행 안에서 열(column)** 을 하나씩 이동
👉 `j = 0 → 1 → 2`

---

## 왜 `arr3.length`를 쓰면 안 될까?

만약 이렇게 쓰면:

```js
for (let j = 0; j < arr3.length; j++) {
```

지금은 **우연히** 잘 돌아가지만 ❌
이유는…

```js
const arr3 = [
  ['A','B'],
  ['C','D','E'],
  ['F']
];
```

이런 배열이면?

* `arr3.length` → 3
* `arr3[0].length` → 2
* `arr3[1].length` → 3
* `arr3[2].length` → 1

👉 **각 행의 길이가 다름**

그래서 **항상**

```js
arr3[i].length
```

를 써야 안전함.

---

## 한 줄로 정리

| 코드               | 의미          |
| ---------------- | ----------- |
| `arr3.length`    | 행 개수        |
| `arr3[i]`        | i번째 행 (배열)  |
| `arr3[i].length` | i번째 행의 열 개수 |

---

## 비유로 이해하면

📦 책장(arr3)
📚 한 칸(arr3[i])
📄 책 개수(arr3[i].length)
